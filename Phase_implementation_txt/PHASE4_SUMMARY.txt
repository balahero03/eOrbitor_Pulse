                            API TESTING
================================================================================

GET /api/quotations?page=1&limit=20&status=DRAFT&search=qt-
- Returns: List of quotations with pagination
- Status: 200 OK
- Includes: customer, deal, totals, dates

POST /api/quotations
- Creates: New quotation with products
- Returns: Created quotation object
- Status: 201 Created
- Validates: dealId, customerId, items required
- Calculates: subtotal, tax, total automatically

GET /api/quotations/:id
- Returns: Full quotation detail
- Status: 200 OK
- Includes: items, customer, deal, orders, metadata

PATCH /api/quotations/:id
- Updates: Quotation fields (items, notes, status)
- Returns: Updated quotation
- Status: 200 OK
- Recalculates: subtotal, tax, total
- Increments: revision number

DELETE /api/quotations/:id
- Deletes: Quotation
- Returns: Success message
- Status: 200 OK

POST /api/quotations/:id/send
- Sends: Quotation to customer
- Returns: Updated quotation
- Status: 200 OK
- Sets: status = SENT, sentAt = now

POST /api/quotations/:id/approve
- Approves: Quotation
- Returns: Updated quotation
- Status: 200 OK
- Sets: status = ACCEPTED, approvedAt = now, approvedById = userId

GET /api/products?search=license&category=software
- Returns: List of active products
- Status: 200 OK
- Includes: sku, name, basePrice, tax, inventory

================================================================================
                        IMPLEMENTATION HIGHLIGHTS
================================================================================

TAX CALCULATION SYSTEM:
- Product model stores tax percentage
- Each quotation item uses product's tax rate
- Line-level calculation: (Quantity × Price) + ((Quantity × Price) × Tax%)
- Aggregated in quotation totals
- Supports variable tax rates (5%, 12%, 18%)

DYNAMIC PRODUCT SEARCH:
- Real-time search as user types
- Fetches from database (no hardcoded list)
- Shows product SKU, name, price, tax
- Allows editable prices per quotation

QUOTATION NUMBERING:
- Auto-generated format: QT-YYYY-00001, QT-YYYY-00002
- Unique constraint prevents duplicates
- Easy for customers to reference

REVISION TRACKING:
- Version history without duplication
- Each update increments revision
- Allows rollback to previous versions (future)

WORKFLOW MANAGEMENT:
- Status transitions: DRAFT → SENT → ACCEPTED
- Alternative: DRAFT → SENT → REJECTED
- Timestamps recorded for each transition
- User tracking (approvedById)

REAL-TIME CALCULATIONS:
- Frontend calculates as user edits
- Backend validates and recalculates on save
- Both ensure data consistency
