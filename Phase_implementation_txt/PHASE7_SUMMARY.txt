================================================================================
                            TECHNICAL STACK
================================================================================

BACKEND:
- API Route: Next.js 14 App Router
- Database: Prisma ORM with PostgreSQL
- Authentication: JWT token validation
- Data Validation: Server-side validation
- Soft Deletes: isActive boolean flag (no hard deletes)
- Unique Constraints: SKU and GST number uniqueness
- Relationships: Product → Inventory, Product ← VendorProduct → Vendor
- Error Handling: Proper HTTP status codes and error messages

FRONTEND:
- Framework: React 18 with Next.js 14
- Language: TypeScript for type safety
- Styling: Tailwind CSS
- State Management: React useState/useEffect hooks
- Currency Formatting: Intl.NumberFormat (INR)
- Forms: HTML5 with client-side validation
- Dynamic Selectors: Categories auto-extracted from products
- Color Coding: Status badges with contextual colors
- Links: Internal navigation and external URLs

DATABASE SCHEMA (already existed in schema.prisma):
- Product: sku (unique), name, category, description, basePrice, tax, isActive
- Inventory: productId (unique), quantity, reorderLevel, warehouseLocation, lastRestockDate
- Vendor: vendorName, gstNumber (unique), email, phone, website, paymentTerms, rating, isActive
- VendorProduct: vendorId + productId (unique composite), vendorSku, vendorPrice, leadTime, minimumOrder
